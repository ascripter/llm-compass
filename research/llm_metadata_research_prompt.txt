<data_schema>
NOTE: Fields contain a type hint in brackets and "may be empty" in case information may not exist. If you see "conditionally required" pay attention to the field description when this field may be empty and when you should find values.
* **name_normalized:** (string) normalized canonical name of the LLM: remove date suffixes for the common release versions. Keep them for preview, experimental, snapshot etc. versions
* **name_aliases:** (string representing comma-separated list) Alternative names under which the EXACTLY same model build can be found. DO NOT add names of other variants of the same model, only identifiers that are usually routed to the exactly same API endpoint as `name_normalized`.
* **model_type:** (enum: "base", "instruct", "thinking", "generator") Defines the core training paradigm. Use "base" for foundational completion models without chat tuning. Use "instruct" for models fine-tuned for chat and instruction following (e.g., GPT-4o, Claude 3.5 Sonnet, Llama-3-Instruct). Use "thinking" for models trained via RL to output internal reasoning traces (e.g., o1, o3-mini, DeepSeek-R1). Use "generator" for models that primarily generate media (or transcribe media to text) and are not conversational (e.g., Midjourney, Sora, Flux, Whisper).
* **provider:** (string) name of the provider
* **release_date:** (date in YYYY-MM-DD format) release date of the model (most recent stable version). If the exact day of release is unknown, fall back to the first of the month.
* **parameter_count**: (number string with suffixes "T" for trillion, "B" for billion, "M" for million, may be empty) number of parameters (if known, otherwise leave empty)
* **architecture:** (string; tags/keywords, may be empty) description for the specific, detailed architecture of this model (as far as it's known)
* **available_quantizations:** (string, may be empty) if you find quantized versions of a model smaller than 16 bit, enter a list comma-separated values that represent quantizations (i.e. "GGUF", "q4_k_m", "fp8", "int4"), otherwise leave blank
* **distillation_source:** (string, may be empty) distillation source (if applicable): Research whether this model has a larger "master" model from which it was derived / distilled. Fill ONLY if the other model name can really be considered as a distillation parent, NOT a previous or sibling version.
* **modality_input:** (comma-separated values out of: "text", "image", "video" and "audio") modalities of allowed inputs
* **modality_output:** (comma-separated values out of: "text", "image", "video" and "audio") modalities of possible outputs
* **context_window:** (int, conditionally required) size of the context window in tokens (also for image, video and audio models if data is available). Required for models whose major purpose is text input.
* **max_output_tokens:** (int, conditionally required) the maximum number of output tokens for a single chat iteration / a single query. Required for models whose major purpose is text output.
* **cost_input_text_1m:** (float, conditionally required) Approximate base cost (in USD) per 1M input tokens (ignore caching discounts). For proprietary models, pick the best offer. For open-weights, use the cheapest major aggregator (e.g., OpenRouter) for an unquantized variant. If the original currency is not USD, calculate the exchange rate and log the original price in raw_cost_notes_input. If input tokens are not charged, enter 0.00.
* **cost_output_text_1m:** (float, conditionally required) Same rules as cost_input_text_1m, but for output tokens.
* **cost_input_image_1024:** (float, conditionally required) Approximate cost (in USD) per single 1024x1024 input image. Apply aggregator/USD rules from cost_input_text_1m. If priced per token, calculate the exact token-to-image conversion for THIS model  (prefer standard / medium quality and 1024 x 1024 pixels if you encounter several choices). DO NOT guess—if the conversion rate isn't explicitly documented or mathematically certain, leave this float field blank ("") and put the raw pricing details in raw_cost_notes_input instead.
* **cost_output_image_1024:** (float, conditionally required) Same rules as cost_input_image_1024, but for output images.
* **cost_input_audio_1h:** (float, conditionally required) Approximate cost (in USD) per 1 hour of input audio. Apply aggregator/USD rules from cost_input_text_1m. If priced per token, calculate exact conversion per hour audio. DO NOT guess—if uncertain, leave blank ("") and use raw_cost_notes_input.
* **cost_output_audio_1h:** (float, conditionally required) Same rules as cost_input_audio_1h, but for output audio.
* **cost_input_video_1s:** (float, conditionally required) Approximate cost (in USD) per 1 second of HD video (1280x720). Apply aggregator/USD rules from cost_input_text_1m. If priced per token, calculate exact conversion for HD video (1280 x 720 px) seconds. DO NOT guess—if uncertain, leave blank ("") and use raw_cost_notes_input.
* **cost_output_video_1s:** (float, conditionally required) Same rules as cost_input_video_1s, but for output video.
* **raw_cost_notes_input:** (string, may be empty) If any input cost data doesn't match the expected standard (text not in tokens, image not by resolution, audio/video not by duration, or non-USD currency), enter the raw pricing information you found here (e.g., "1.50 / 1m tokens @ 1234 tokens per image" or "15.00 EUR / 1m tokens").
* **raw_cost_notes_output:** (string, may be empty) Same rules as raw_cost_notes_input, but for output pricing.
* **speed_tps:** (int, may be empty) Output generation speed (decode tokens per second; approximate value, i.e. take median from benchmark sites that will give a speed representative for a production environment. Prefer speed based on the provider's official API benchmarks). For models that output only image/video natively, leave this empty. 
* **speed_class:** (enum: "slow", "medium", "fast") speed / latency taken from benchmarks. Rule: less than 40 speed_tps is slow, between 40 and 100 is medium, above 100 is fast. If tps is not available, translate verbal descriptions you find that indicate whether this model is "fast", "medium" or "slow". Take info from benchmark sites that will give a speed representative for a production environment.
* **is_open_weights:** (strict enum: "yes" or "no" ONLY) Is it an open weights model? Use "yes" even if there are commercial restrictions, as long as the weights can be downloaded.
* **license:** (string, may be empty) under which license the model may be used (relevant for open weights models, otherwise may be empty)
* **reasoning_type:** (enum: "none", "standard", "native cot") Defines the model's architectural reasoning paradigm. Use "none" for pure image/audio generators. Use "standard" for standard autoregressive models that reason only when explicitly prompted (e.g., GPT-4o, Claude 3.5 Sonnet, Llama 3). Use "native cot" for dedicated reasoning models trained via RL to automatically generate extended internal thinking before answering (e.g., OpenAI o1/o3, DeepSeek-R1, QwQ, Gemini 2.0 Flash Thinking).
* **tool_calling:** (enum: "none", "standard", "agentic") Defines the tool-calling capabilities of the model. Use "none" for base/foundation models (non-instruct), older models, or pure image/audio/video generators. Use "standard" for models that support basic JSON schema function calling (e.g., standard Llama 3 Instruct, Mistral, most open-weights). Use "agentic" for models with built-in native tools (like native code execution, automated web search, or file parsing handled by the model itself) or high-level agentic capabilities like Computer Use (e.g., Claude 3.5 Sonnet, GPT-4o, Gemini 2.5 Pro).
</data_schema>
